Why locreport

Most line counters treat every line the same.

So a codebase looks like it does nothing but grow. Yet adding tests, comments and docs while shrinking the application code means less to maintain, not more — and one big number can't tell those apart. locreport separates the code you actually maintain from the tests, config, docs and data around it, so you can see which one is really moving.

  1. Appthe actual application code
  2. Testsspecs, fixtures, e2e
  3. Configyaml, toml, manifests
  4. Docsmarkdown, rst
  5. Datajson, csv, xml

Build files and vendored / generated code (node_modules, lockfiles, minified bundles) are detected and excluded from the counts.

Click a legend entry to isolate it; click again to restore all.
